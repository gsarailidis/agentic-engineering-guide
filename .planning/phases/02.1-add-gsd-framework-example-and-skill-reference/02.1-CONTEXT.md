# Phase 02.1: Add GSD Framework Example And Skill Reference - Context

**Gathered:** 2026-03-20
**Status:** Ready for planning

<domain>
## Phase Boundary

This phase adds one dedicated GSD section to the guide as a concrete workflow-framework example layered on top of the local-agent patterns established in Phase 2. It must explain what GSD is, how it fits into the local-agent environment model, and include a categorized skill reference based on the user-provided list. It should deepen the guide with one concrete framework example without turning the document into GSD product documentation or adding a separate cheat sheet.

</domain>

<decisions>
## Implementation Decisions

### Section placement
- The GSD material should appear as one dedicated section in the guide body.
- It should read as a concrete follow-on to the concepts established in Phase 2, not as a detached appendix.

### Depth of explanation
- Explain GSD clearly as a workflow framework built on top of a local tool-using agent environment.
- Use the skill list as the main concrete detail.
- Include both a full categorized skill reference and a shorter guided walkthrough of the main skills.
- Do not expand into deep internal documentation or an implementation tutorial.

### Skill-list organization
- The final skill-list structure does not need to preserve the user's exact grouping if a better editorial grouping reads more clearly.
- Claude may reorganize the skill list for readability as long as the skills and their descriptions remain accurate and easy to scan.

### Comparative framing
- Direct comparison to Claude Code or Codex CLI is allowed only when it adds real explanatory value.
- Comparison should clarify how GSD sits as a workflow layer on top of the environment-level patterns, not turn into a product comparison section.

### Exclusions
- Do not include a "when to use which skill" cheat sheet.
- Do not let the GSD section take over the broader guide.
- Do not turn the section into framework marketing or exhaustive operator documentation.

### Claude's Discretion
- Exact section title for the GSD section
- Exact ordering and grouping of the skill reference inside that section
- How tightly the GSD section connects back to artifacts, workflows, delegation, and verification from Phase 2

</decisions>

<specifics>
## Specific Ideas

- GSD should be presented as a concrete example of what happens when reusable workflows, artifacts, planning state, validation, and execution control are made explicit.
- The section should make clear that GSD is one framework-level realization of the patterns the guide has already defined.
- The skill list should stay readable and reference-like, not become a long operational manual.
- A short guided walkthrough should highlight roughly the main 7-14 skills before the full reference.

</specifics>
